Worker Pool

This post compiles my attempt to create a simple worker pool

Motivation

I am building the testing infrastructure for an internal service at Kaytu. To test one single unit of this service, I would need to perform the following steps:

  • provision the test infrastructure
  • run the service
  • test the outcome
  • destroy the test infrastructure

The problem arises when I calculate that I would need to do this for 150+ times, with their own different parameters and configurations. I am trying to find an efficient way of conducting these tests.

Research

While working with the team, I have come across multiple iterations of the worker pool nodes, the open source ones are as follows:

Project Repository Comments
Kaytu CLI kaytu the plugin interface, amongst other components, exposes a worker pool as part of the sdk.
Plugin AWS plugin-aws involves implementing the above sdk, including pushing jobs to worker pool.
Plugin GCP plugin-gcp an implementation similar to above
Azure Describers to execute jobs pertaining to populating elasticsearch instance with azure resource data

Prerequisites

I am using Go to create my worker pool, mainly because the workers I am testing are in Go, and the third party libraries I am using are in Go.
